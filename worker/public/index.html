How it works

Three steps to a complete DSA portfolio

No servers. No databases. No accounts. Just an extension, your API keys, and your GitHub repo.

1

Install the extension

Load it unpacked from GitHub or install from the Chrome Web Store. Connect your GitHub account via OAuth — we only request repo scope.

2

Add your API keys

Paste your own Gemini, OpenAI, or other AI keys into Settings. Keys stay in your browser's local storage — they never leave your machine.

3

Solve problems as usual

Submit an accepted solution on LeetCode, GFG, or Codeforces. CodeLedger automatically commits it to your repo with metadata and an optional AI review.

Features

Everything you need, nothing you don't

Plugin-based architecture — add platforms, AI providers, and git hosts without touching core code.

⚡

Atomic git commits

Uses the GitHub Tree API to commit code, metadata, and AI review in a single atomic operation. No partial states.

🤖

AI code review

Gemini, OpenAI, Claude, DeepSeek, or local Ollama. Round-robin key pools with automatic fallback if one provider fails.

📊

Analytics dashboard

Heatmap, topic radar, difficulty donut, solve velocity — all built from your local IndexedDB. No telemetry unless you opt in.

🔌

Plugin architecture

Adding a new platform or AI provider is a single file. The BasePlatformHandler contract makes contribution straightforward.

🦊

Chrome + Firefox

A single codebase works on both browsers. No webextension-polyfill — a lightweight compat shim handles the differences.

🗺️

Knowledge graph

Problems are linked by topic and algorithmic pattern. Visualize your progress as a graph and spot gaps in your preparation.

🔑

Bring your own everything

Your solutions live in your GitHub repo. Your AI keys stay in your browser. We run exactly one server — the OAuth relay — and it never sees your code or your keys.
